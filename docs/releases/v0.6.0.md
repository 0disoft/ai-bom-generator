## Highlights

- Add bounded Pipenv `Pipfile.lock` specification 6 parsing with exact versions,
  hashes, extras, markers, indexes, Git references, remote files, and local paths.
- Add a compatibility corpus generated by exact Pipenv 2023.12.1 and 2026.6.2
  releases and require it in source distributions.
- Add deterministic property-style coverage for malformed JSON shapes and
  package evidence combinations.

## Compatibility

Pipenv support is additive and remains explicit through `[[dependencies]]` or
the exact `Pipfile.lock` filename. Existing dependency formats keep their
current behavior, dependency resolution and downloads remain disabled, and
selected Pipenv groups are never inferred.

## Rollback

Published releases are immutable. Pin consumers to `v0.5.0` and publish a
corrective patch if rollback is required.
