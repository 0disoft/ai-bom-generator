## Highlights

- Relative GitHub Action output paths now resolve from the workspace and retain
  verified summary-derived outputs.
- Output commits lock every generated destination, preventing overlapping runs
  with different manifest paths from racing.
- Artifact discovery uses one top-down walk and prunes universally excluded
  subtrees before descent.

## Compatibility

This patch preserves the Python 3.12 through 3.14 runtime range, CLI flags,
GitHub Action inputs and outputs, config schema v1, exporter formats, warning
codes, and exit-code meanings. Persistent destination-adjacent lock files are
internal coordination state outside the target model directory.

## Rollback

Revert the v0.6.1 patch commit and repoint the mutable `v0` Action tag to
`v0.6.0` only after external exact-version smoke verification confirms the
rollback target. Existing generated BOM files remain consumable because output
schemas and exporter contracts are unchanged.
